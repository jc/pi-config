Use this skill when the task is a complex feature, a significant refactor, or has ambiguous requirements that benefit from a living plan.

## Source of truth

The ExecPlan standard is defined in `reference/PLANS.md`

## Workflow

### Create the ExecPlan

When asked by the user:

1. Read the full reference `PLANS.md`
2. Create a single ExecPlan file as instructed (formatting rules matter).
3. Request the user review and provide comment. Iterate accordingly.

### Implement an ExecPlan

When asked by the user:

1. Read the full reference `PLANS.md`
2. Read the ExecPlan provided by the user
3. Follow the ExecPlan and process as instructed.
